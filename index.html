Deity's Vial

A Javascript, browser, 3D MMORPG

This game has a Lazy Loader for downloading assets from a CDN, merging hundreds of thousands of vertices, and adding a new chunk to the world map, as the player moves.

Loading was asynchronous and geometry merging and re-indexing was done into a thread (Web Worker).
The Pixel Art style, as well as the low poly 3D models, were chosen to improve loading time, otherwise loading screens would have been necessary.

The interface was done in HTML5 + CSS3.
The only library used was Three.js.

The project had only 6 months of work and was left unfinished,
having a Java, non-blocking io sockets, server-side for login, character selection, items DB, etc...

MauaEureka

An Android App, made for a university fair

This app had a login screen, using SOAP protocol, only allowing visitors of the fair to access it.

The main goal was to orient people attending to this event. The app contained a map of the booths, a graph to trace a route from the user location to the desired booth, a GPS with the map of the university, and a voting system for the projects in exposition.

A heat map was generated after the fair, using the users locations stored into a MySQL DB.

I was given 1 month to create this app, so the UI was left unnatended.

Bau Slider

A game made in Unity

This game was my first experience with Unity, and the main goal was to learn the DO's & DON'Ts of the Unity Engine.

All the models, textures, songs and codes were produced by me. I was trying to achieve the old Nintendo64 feel and trying to make the asset production faster as well.

It uses both the Facebook, and the Google Play API's. For setting high scores and having a look at your friends achievements. A MySQL DB was used to keep track of the user items.

The project was concluded in around 5 months.

The songs composed for the game may be found on my bandcamp account.
